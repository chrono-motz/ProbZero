# ProbZero

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.18701250.svg)](https://doi.org/10.5281/zenodo.18701250)

### A Full-Search Reinforcement Learning Framework

ProbZero is an AlphaZero-inspired reinforcement learning framework that attempts to maximize utilization of expensive search computations by learning from entire search trees rather than only compressed root-level search outcomes.

> **Note:** ProbZero is an experimental framework exploring full-search supervision and alternative policy/value formulations. It does not claim superiority to AlphaZero without direct empirical comparisons, but serves as a testbed for novel reinforcement learning search architectures.

---

## Key Ideas

### Full-Search Supervision

For a typical Othello game, AlphaZero may perform tens of thousands of search expansions but ultimately trains on a highly compressed summary of that search. 

ProbZero attempts to recover training signal from all explored search states rather than only root-level outcomes.

This allows training to leverage thousands of explored search states generated during self-play rather than only a compressed root-level summary.

---

### Why Full-Search Learning Requires New Targets

Simply applying full-tree supervision to AlphaZero is problematic.

AlphaZero policies imitate visit-count distributions generated by search. When supervision is extracted from every explored node, exploratory actions receive training credit even when they are known to be suboptimal.

ProbZero addresses this by predicting optimal-move probabilities derived from search value estimates rather than visit-count distributions.

---

### Outcome–Value Decoupling

Learning from complete search trees introduces stability challenges.

Directly propagating value estimates across complete search trees can lead to unstable averaging effects. ProbZero therefore separates grounded terminal supervision from generalized value estimation.

ProbZero addresses this using two separate estimators:

- Outcome Estimator
- Value Estimator

The outcome estimator learns directly from terminal trajectories.

The value estimator is bootstrapped from outcome estimates and generalizes to incomplete positions.

This decoupling enables stable tree-wide learning while preserving grounded terminal supervision.

---

### Optimal-Move Probability Learning

AlphaZero policies imitate search visit distributions.

ProbZero instead predicts:

`P(move is optimal)`

derived from search value estimates.

This removes the dependence between policy learning and exploration behavior.

Benefits:

- Full-tree supervision becomes feasible.
- Exploration noise does not directly corrupt targets.
- Search no longer needs visit-count based objectives.

---

### Probabilistic Parallel Search

Because policy learning is based on optimal-move probabilities rather than visit distributions, search can be reformulated.

ProbZero introduces a probabilistic search framework that supports massively parallel batched inference.

- **Eliminates Dirichlet-noise injection**.
- **Virtual Loss**: Because search is probabilistic rather than greedily selecting a single best path, concurrent expansions naturally diversify search trajectories, reducing the need for virtual-loss style coordination.
- **Replay Buffers**: The current implementation operates without replay buffers, relying instead on probabilistic search and dense full-tree supervision for data diversity.

Search volume is distributed probabilistically through the tree instead of following traditional sequential MCTS expansion.

---

## Comparison with AlphaZero

| Component | AlphaZero | ProbZero |
| :--- | :--- | :--- |
| **Training Signal** | Root Only | Full Tree |
| **Policy Target** | Visit Counts | Optimal-Move Probability |
| **Value Learning** | Single Head | Outcome + Value |
| **Search** | Sequential MCTS | Probabilistic Parallel Search |
| **Virtual Loss** | Required | Not Used |
| **Replay Buffer** | Common | Not Used (Current Implementation) |

## Technical Architecture

### Target Generation Flow

```text
       Full Search Tree
              ↓
      Outcome Estimator
              ↓
       Value Estimator
              ↓
 Optimal-Move Probabilities
        ├── Policy Targets
        └── Probabilistic Search
```

### Neural Network Architecture (Dynamically Sized)

```text
Input: 2 × N × N (self pieces, opponent pieces)
  ↓
Conv2d(2 → 50, 3×3) + BN + ReLU
  ↓
10× ResBlock(50 channels, 3×3)
  ↓
┌─────────────────────┬──────────────────────┬──────────────────────┐
│     Policy Head     │      Value Head      │    Outcome Head      │
│  Conv(50→32,1×1)    │   Conv(50→32,1×1)    │   Conv(50→32,1×1)    │
│  FC(→ N×N+1)        │   FC(→ 50 → 3)       │   FC(→ 50 → 3)       │
│  → (N×N+1) logits   │   → 3 (L,D,W)        │   → 3 (L,D,W)        │
└─────────────────────┴──────────────────────┴──────────────────────┘
```

## Experimental Results

### 4×4 Othello

The strongest current validation.

After approximately 20 self-play iterations:

- Achieved optimal play on solved 4×4 Othello.
- Defeated a perfect-play engine as the second player.
- Correctly predicted game-theoretic outcomes throughout complete games.

This demonstrates that ProbZero can recover optimal behaviour using the proposed training framework.

---

### 6×6 Othello

Training completed for approximately 80 iterations.

Results remain preliminary and require further evaluation.

## Quick Start

### 1. Requirements
- **LibTorch C++ (1.13+)**: Download the C++ version of PyTorch for your system/CUDA version.
- **Python**: PyTorch installed (`pip install torch`)
- **CMake**: Build system

### 2. Build C++ Engine
Set `CMAKE_PREFIX_PATH` to your LibTorch directory.

```bash
mkdir -p build && cd build
cmake .. -DCMAKE_PREFIX_PATH=/path/to/libtorch
make -j$(nproc)
cd ..
```

### 3. Run Training
This script manages the self-play → training loop automatically, and will prompt you to pick a board size (e.g. 4 or 6).

```bash
python scripts/run.py
```
Alternatively, specify the board size explicitly:
```bash
python scripts/run.py --size 4
```
